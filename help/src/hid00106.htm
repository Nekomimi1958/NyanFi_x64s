用語について

タブ
表示メニューまたは ShowTabBar コマンドでタブバーを表示できます。
タブは、一般的なタブ型ファイラーとは異なり、左右のディレクトリを組として切り替えるためのボタンとして機能します。
コマンド
ほとんどの機能は、コマンドで処理されます。
コマンドは、キーやツールバーなどに割り当てられます。また、InputCommands コマンドの入力ボックスからも実行できます。
コマンドファイル
NyanFi のコマンドを記述した、テキストファイルです。バッチファイルに似た文法によって、一連の処理を行えます。
コマンドファイルは、ExeCommands コマンドなどで実行できます。
ワークリスト
いろいろな場所にあるファイルやディレクトリをまとめた参照リストです(→ ワークリスト)。
よく使うファイルやディレクトリを登録しておくと素早くアクセスできます。
WorkList コマンドで仮想リストとして開けます。
タグ
ファイルやディレクトリには、AddTag コマンドでタグ(ラベル)を付けて管理することができます(→ タグによるファイル管理)。
これは、「プロパティ」で設定できるタグとは無関係です。また対象そのものには一切変更を加えません。
タグに関するデータは、NyanFi 実行ディレクトリの TAGDATA.TXT というファイルに保存されています。
栞マーク
ファイルやディレクトリには、Mark コマンドで栞マークを設定できます。その際にメモを付けることも可能です。
MarkList コマンドで栞マーク一覧を開くと、栞マークが設定されている項目を確認できます 。
栞マークに関するデータは、INIファイルに保存されます。
ディレクトリ
Windows では一般的に「フォルダ」と呼ばれますが、NyanFi はエクスプローラのようなシェル拡張系のファイラーではないため、ディレクトリと呼んでいます。ディレクトリの中身はファイル/ディレクトリだけですが、フォルダにはファイルではないオブジェクトが含まれている場合があります。
ファイル名主部
パス無しファイル名から拡張子を除いた部分です。一般的には「ベース名」と呼ばれていますが、拡張子を含む場合もあり、曖昧性を避けるため NyanFi ではファイル名主部と呼んでいます。
例えば "D:\NyanFi64s\NyanFi.exe" というファイル名の場合、ファイル名主部は "NyanFi" になります。
仮想ディレクトリ/リスト
アーカイブファイルは、仮想的なディレクトリのように開いて扱うことができます。
また、ワークリスト、検索結果リスト(→ SaveAsResultList/LoadResultList)、検索設定ファイル(→ SaveAsFindSet/LoadFindSet)なども、仮想的なファイルリストとして表示できます。
代替データストリーム
Windows の NTFS の機能のひとつで、通常のファイルやディレクトリに追加データを格納できます。
NyanFi では、サムネイルキャッシュなどで利用しています(→ NyanFi が用いるファイル)。
OpenADS コマンドで仮想リストとして開き、コピー、削除、閲覧などを行えます。
